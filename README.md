# 🛡️ Bilibili 全能护盾 (Bilibili Omnipotent Shield)

> **可能是目前最强力、最智能、最稳定的 B 站深度净化脚本。**
> 拒绝“信息流污染”，还你一个只有 UP 主投稿、纯净清爽的 Bilibili。

## 📖 简介 (Introduction)

**Bilibili 全能护盾** 不仅仅是一个简单的广告拦截插件，它是一个运行在浏览器端的“内容防火墙”。

它采用 **API 数据劫持 (Fetch/XHR Hook)** 与 **DOM 元素清洗** 双核驱动技术。在数据到达浏览器渲染层之前，脚本就已经识别并剔除了其中的广告、推广、营销号、综艺板块以及各类垃圾信息。

**核心理念：** 看不见的拦截，才是最好的拦截。零报错，零闪烁，全静音。

## ✨ 核心功能 (Key Features)

### 1. 🧬 核弹级 API 净化 (Deep API Cleaning)

* **网络熔断**：直接阻断广告 SDK (`cm.bilibili`) 和数据追踪接口的请求，返回伪造的空响应，从根源节省流量并阻止广告加载。
* **户口核查**：深入解析 API 数据结构，识别视频的 `tname` (分区) 和 `goto` (跳转类型)。凡是“综艺”、“课堂”、“付费课程”、“游戏推广”，无视标题，直接在数据层抹除。
* **智能伪造**：拦截数据的同时，自动填充合法的空数据结构，防止 B 站前端框架因数据缺失而报错崩溃。

### 2. 🚫 营销号与烂梗终结者

* **MCN 识别**：内置强大的正则库，自动识别并拉黑“xx新闻”、“xx视频”、“xx文旅”、“xx讲故事”等流水线机构账号。
* **初始号清洗**：自动屏蔽 `bili_12345678` 这类未修改昵称的机器号/尸体号。
* **黑话拦截**：精准识别并拦截评论区和标题中的擦边黑话（如“馒头币”、“航班起飞”等）。
* **卖课拦截**：秒杀标题中带有“送教程”、“免费领”、“变现实操”的引流视频。

### 3. 🛡️ 评论区深度治理

* **反烂梗**：自动过滤低智阴阳怪气言论。
* **纯@过滤**：自动隐藏评论区中只有 `@某人` 而无实质内容的“互暖”评论。
* **兼容性**：独家支持 **Bilibili-Old** 的评论区加载方式 (XHR 劫持)。

### 4. 🎨 UI 融合与旧版复刻

* **可视化控制台**：页面右下角驻留半透明盾牌图标，点击即可呼出控制面板，实时开关各项功能。
* **样式复刻**：内置“粉色顶栏”样式注入引擎，一键找回当年的 B 站视觉体验。
* **完美共存**：UI 设计深度适配 `MotooriKashin/Bilibili-Old`，按钮自动垂直堆叠，互不冲突。

### 5. 🔇 智能错误抑制

* 脚本接管了浏览器的 `console.error`，自动识别并拦截因去广告导致的红字报错。您的控制台将始终保持清爽绿色。

## 📥 安装与使用 (Usage)

### 安装

1. 安装浏览器扩展 [Tampermonkey (油猴)](https://www.tampermonkey.net/)。
2. 新建脚本，将 `V1.1.0` 版本的代码完整复制粘贴并保存。

### 控制面板

安装成功后，刷新 B 站页面，右下角会出现一个 **🛡️** 图标。点击即可打开控制面板：

* **📺 强制旧版样式**：一键开启粉色顶栏怀旧模式。
* **💉 Bilibili-Old 兼容模式**：如果您安装了旧版 B 站脚本，请开启此项以确保评论区过滤生效。
* **网络熔断 (去广告)**：建议常开，从网络层阻断广告。
* **首页楼层净化**：建议常开，清除首页写死的综艺/推广板块。
* **拦截综艺/娱乐区**：不想看综艺？关掉它，从此你的世界只有 UP 主。
* **拦截营销号/初始号**：自动清理时间线上的垃圾账号。

如果您懂代码，可以在脚本头部的 `CONFIG` 和 `CONSTANTS` 区域进行深度定制：

```javascript

    "综艺", "课堂", "推广", "直播" // ...
];

```

## ❓ 常见问题 (FAQ)

**Q: 为什么启用了脚本，有时候首页还是会闪现一下综艺楼层？**
A: 这是因为 B 站首页的部分内容是 SSR（服务端渲染）直接写在 HTML 里的。脚本需要等页面加载后立即执行 DOM 清洗。虽然我们已经做到了毫秒级响应，但在性能较差的机器上可能会捕捉到一瞬间的残影，随后它会被立即移除。

**Q: 开启脚本后，控制台看不到报错了，是 B 站修复了吗？**
A: 不，是脚本拦截了报错。为了防止因拦截广告导致 B 站前端逻辑崩溃抛出大量红字，脚本内置了“错误抑制系统”，只拦截特定的无害报错，不影响正常调试。

**Q: 这个脚本会封号吗？**
A: 不会。脚本的所有操作均运行在您的本地浏览器中。只有当触发“自动拉黑营销号”功能时，脚本会调用 B 站官方的拉黑接口（这等同于您手动点击了拉黑），属于正常用户行为。
