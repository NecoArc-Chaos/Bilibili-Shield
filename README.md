# 🛡️ Bilibili 全能护盾 (Bilibili Omnipotent Shield)

> **一款基于浏览器端的 B 站深度定制与净化工具。**
> 旨在通过技术手段过滤信息流中的无关内容，还原纯净的视频浏览体验。支持 Bilibili-Old 旧版样式。

## 📖 简介 (Introduction)

**Bilibili 全能护盾** 是一款致力于提升 B 站网页端浏览体验的辅助脚本。

针对当前网页端存在的广告加载、非偏好内容推荐、以及评论区冗余信息，本脚本采用 **API 数据预处理 (Fetch/XHR Hook)** 与 **DOM 元素管理** 技术，在数据渲染前进行筛选与优化，帮助用户专注于高质量的 UP 主投稿内容。

**设计理念：** 无感运行，静默过滤，高度可配置。

## ✨ 核心功能 (Key Features)

### 1. 🧬 深度 API 过滤 (Deep API Filtering)

* **网络请求优化**：拦截部分商业推广 SDK 及数据追踪接口的请求，返回标准空响应，从而节省网络资源并提升页面加载速度。
* **内容筛选**：深入解析 API 返回的数据结构，根据用户设定的规则（如分区 `tname`、跳转类型 `goto`），自动移除“综艺”、“课堂”、“付费课程”及“游戏中心”等非视频类内容。
* **异常处理**：在过滤数据的同时，自动填充合法的空数据结构，防止因数据缺失导致的前端页面渲染错误。

### 2. 🚫 账号与内容管理

* **机构账号识别**：内置匹配规则，支持识别并屏蔽具有特定命名特征的流水线式机构账号（如部分同质化严重的新闻、营销、搬运账号）。
* **异常账号过滤**：自动隐藏未修改默认昵称（如 `bili_12345678`）的初始账号产生的互动内容。
* **关键词屏蔽**：支持自定义屏蔽标题中包含特定营销词汇（如“免费教程”、“实操变现”等）的引流视频。
* **特定词汇过滤**：支持识别评论区或标题中的特定隐晦用语（如谐音梗、黑话），净化社区环境。

### 3. 🛡️ 评论区深度治理

* **低质内容折叠**：自动识别并隐藏无实质意义的刷屏评论、重复玩梗评论。
* **无效互动过滤**：自动隐藏仅包含 `@某人` 而无实质内容的“互暖”或“占楼”评论。
* **旧版兼容**：独家支持 **Bilibili-Old** 脚本的评论区加载方式 (XHR 劫持)，确保在旧版 UI 下依然生效。

### 4. 🎨 UI 融合与旧版复刻

* **可视化控制台**：页面右下角提供半透明悬浮图标，点击即可呼出配置面板，实时开启/关闭各项功能。
* **经典样式复刻**：内置样式引擎，可一键将顶栏还原为经典的粉色主题，找回熟悉的使用体验。
* **组件兼容**：UI 设计深度适配 `MotooriKashin/Bilibili-Old`，按钮位置自动堆叠，互不遮挡。

### 5. 🔇 智能错误抑制

* 脚本接管了浏览器的控制台输出，自动识别并拦截因请求阻断而产生的红字报错信息，保持开发者工具控制台的整洁。

## 📥 安装与使用 (Usage)

### 安装步骤

1. 安装浏览器扩展 [Tampermonkey (油猴)](https://www.tampermonkey.net/)。
2. 新建脚本，将 `V1.1.0` 版本的代码完整复制粘贴并保存。

### 功能控制

安装成功后，刷新 B 站页面，右下角会出现一个 **🛡️** 图标。点击即可打开控制面板：

* **📺 强制旧版样式**：开启粉色顶栏怀旧模式。
* **💉 Bilibili-Old 兼容模式**：如果您安装了旧版 B 站脚本，请开启此项以确保评论区过滤生效。
* **网络请求优化**：建议常开，阻断非必要请求。
* **首页版面净化**：建议常开，移除首页固定的推广板块。
* **拦截特定分区**：可选择是否隐藏综艺、娱乐等非核心分区。
* **拦截机构/初始号**：自动清理时间线上的低质量账号内容。

## ⚙️ 高级配置 (User Config)

如果您具有编程基础，可以在脚本头部的 `CONFIG` 区域进行个性化定制：

```javascript
// 白名单：包含这些关键词的内容将不会被拦截
const WHITE_LIST = [
    "原神", "极客湾", "硬核", "教程", "喜欢的UP主"
];

// 自定义屏蔽词：包含这些词的视频/评论将被隐藏
const KEYWORDS = {
    toxic: ["不想看到的词1", "不想看到的词2"]
};

// 目标分区拦截：从这里移除你想保留的分区
const SECTION_BLOCK_LIST = [
    "综艺", "课堂", "推广", "直播" // ...
];

```

## ❓ 常见问题 (FAQ)

**Q: 为什么启用了脚本，有时候首页还是会短暂显示被屏蔽的楼层？**
A: B 站首页部分内容采用服务端渲染 (SSR)，直接包含在 HTML 中。脚本会在页面加载后立即执行 DOM 清洗。受限于浏览器渲染机制，可能会有极短暂的显示，随即会被移除。

**Q: 开启脚本后，控制台看不到报错了，是 B 站修复了吗？**
A: 这是脚本的“错误抑制”功能在起作用。为了防止因拦截广告导致 B 站前端逻辑抛出大量无意义的报错，脚本自动屏蔽了相关的错误日志。

**Q: 使用本脚本会导致账号被封禁吗？**
A: 不会。脚本的所有操作均运行在您的本地浏览器端，本质上属于页面样式的本地修饰。脚本触发的“自动拉黑”功能调用的是 B 站官方接口，等同于您手动点击了拉黑，属于正常的用户交互行为。
